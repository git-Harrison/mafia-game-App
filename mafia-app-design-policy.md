# mafia-app — 설계·구현 정책 (IP·표절 방지)

이 문서는 `mafia-app-context.md` 와 함께 읽는다. Claude Code 는 코드/UI/문구/명칭을
생성하기 전에 이 문서의 원칙에 어긋나지 않는지 검증해야 한다.

## 0. 한 줄 요약

> 이 게임은 **특정 마피아 게임을 복제하지 않는다.** 전통적인 마피아/웨어울프류
> 소셜 추리 게임의 **일반 룰**만 참고하고, 표현(UI·캐릭터·문구·명칭)은 모두
> 독자 제작한다.

## 1. 사용 가능한 일반 룰 (참고 OK)

전통적 사회 추리 게임의 공통 메커니즘 — 어느 한 게임의 고유 발명이 아닌 장르
표준. 자유롭게 활용해도 무방.

- 낮 / 밤 페이즈 반복
- 시민팀 vs 마피아팀 진영 구도
- 낮 토론 + 투표로 한 명 처형
- 밤에 마피아의 공격
- 경찰(또는 그에 준하는 조사 직업)의 정체 조사
- 의사(또는 그에 준하는 보호 직업)의 사망 방지
- 직업 랜덤 배정 (서버 권위)
- 마피아 전원 제거 → 시민팀 승
- 마피아 수 ≥ 시민팀 수 → 마피아팀 승

## 2. 반드시 피할 것 (Hard No)

다음 항목 중 하나라도 해당하면 **그대로 구현하지 말고 대체안을 제안한다.**

1. **기존 게임의 UI 복사** — 화면 배치, 색상 팔레트, 아이콘 모양, 애니메이션
   타이밍이 특정 게임과 즉시 식별될 정도로 닮은 경우.
2. **기존 게임의 캐릭터/도트/일러스트 리소스 복사** — 픽셀 단위 모사·트레이싱·
   외형 비례를 그대로 가져오는 모든 행위.
   **에셋 소스 (M7 갱신, 2026-05-14):** AI 픽셀아트 (Midjourney / Stable Diffusion /
   DALL-E 등) + 무료 라이선스 팩 (Kenney·LPC·CC0) + CustomPaint 자체 작화 자유 혼합.
   외부 무료 에셋 출처 기록 의무는 폐기. **단 AI 생성 결과물도 §2 hard-no 6개 자가검증 의무 동일하게 적용** — Among Us 우주복 비례 / Town of Salem 단상 시그니처 / 특정 게임 닮은 실루엣이 나오면 prompt 재생성 또는 변주.
3. **기존 게임의 룰 설명문 / 튜토리얼 문구 복사** — "당신은 ~ 입니다", "밤이
   되었습니다" 류의 상징적 카피를 그대로 옮기는 행위. 자체 문장으로 다시 쓴다.
4. **기존 게임의 직업명·스킬명·아이템명·세계관을 그대로 사용** — 예시:
   "Sheriff", "Doctor 가 'heal' 한다", "Mafioso", "Town", "Salem", "Crewmate",
   "Impostor" 등 특정 작품에 강하게 결부된 명칭. 일반어("의사", "경찰", "시민")
   는 장르 표준이라 사용 가능. 단 **고유한 별칭/세계관 명칭은 자체 작명**한다.
5. **Among Us / Town of Salem 등 특정 게임과 혼동될 구조·디자인 모방** —
   캐릭터 실루엣(긴 다리 우주복, 둥근 헬멧 등), 상징적 게임 흐름 화면, 시그니처
   사운드, 보이스 이펙트.
6. **기존 게임명과 유사한 이름** — 음운/철자가 비슷하거나 줄임말이 겹치는 명칭.
   출시 전 상표 검색 (KIPRIS, USPTO, 구글 일반 검색) 으로 충돌 확인.

## 3. 개발 원칙

- **게임명은 고유하게 만든다.** 작명 후보가 나오면 한국·미국·일본·EU 상표 검색
  결과까지 확인하기 전엔 로고/스플래시/스토어 메타데이터에 박지 않는다.
- **캐릭터·맵·UI·아이템·펫·이펙트 — AI 픽셀아트 + 무료 에셋 + 자체 작화 자유 혼합** (M7 갱신).
  특정 인기 게임 (Among Us / Town of Salem 등) 시그니처와 닮음 의심 시 색상·실루엣 변주
  (§2 hard-no 자가검증). 출처 기록은 더 이상 의무 X.
- **룰 설명문과 시스템 메시지는 직접 작성** — 카피는 짧고 평이하게. 다른 게임의
  유명 카피("Among Us!", "Town has spoken" 등) 의 한국어 의역도 금지.
- **메커니즘은 참고하되 표현은 독자적으로** — 같은 메커니즘이라도 화면 구성,
  단계 명칭, 안내 카피는 새로 디자인한다. 예) 공통 메커니즘이 "보호 직업이 밤에
  타인의 집을 방문해 마피아 공격을 무효화" 라면, 이를 어떤 화면으로 보여줄지
  (집 진입 / 보호 부적 / 보호막 그래픽 등) 는 우리 게임만의 표현으로 결정.
- **출시 전 상표 검색 의무** — 앱 이름, 로고, 주요 명칭(직업명·시즌명·아이템명)
  은 스토어 제출 전 상표 검색 + 도메인 가용성 확인. 4단계(스토어 배포 준비) 의
  체크리스트 항목.

## 4. Claude Code 의 행동 규칙

코드/UI/문구/이름을 생성·수정할 때 다음 절차를 따른다.

1. **유사도 자가검증.** 출력 직전에, "이게 어떤 기존 작품을 즉시 떠올리게 하나?"
   를 스스로 묻는다. 떠오르는 게 있으면 §2 의 항목과 대조한다.
2. **위험 신호가 보이면 멈춘다.** 그대로 구현하지 않고, 대신:
   - 어떤 항목에 해당하는지 짧게 명시,
   - 대체안 1~2 개를 제안 (이름·색상·구조·문구 수준에서 충분히 변주),
   - 사용자 결정을 받고 진행.
3. **회색지대는 사용자에게 묻는다.** "이건 일반 룰 vs 특정 게임 고유 표현 중
   어디인가" 가 모호하면 단정하지 말고 사용자에게 판단을 요청.
4. **이미 코드/문서에 들어간 의심 문구·명칭을 발견하면 보고** 한다. 사용자가
   인지한 뒤에야 수정한다.
5. **에셋 출처 기록 의무 폐기 (M7 갱신, 2026-05-14)** — AI 픽셀아트 + 무료 라이선스 에셋
   자유 사용. CREDITS.md 미작성. 단 AI 생성 prompt 는 `client/assets/ai_pixel/prompts.md`
   에 모아두면 비주얼 일관성 + 재생성 편의 ↑ (의무 X, 권장만).

## 5. 위반 사례 vs 안전한 표현 — 예시 모음

| 영역 | 위반 예 | 안전한 대체 |
|---|---|---|
| 캐릭터 디자인 | Among Us 풍 우주복 비례 | 자체 정의한 픽셀 비례·실루엣, 다른 의상 컨셉 |
| 시작 카피 | "마피아가 당신을 노립니다" (특정 작품 카피의 한국어판 그대로) | "어둠 속에서 누군가 움직입니다" (자체 카피) |
| 직업 별칭 | "Sheriff", "Mafioso", "Impostor" | 일반어 그대로 "경찰", "마피아" 또는 자체 별칭 |
| 게임 이름 | "Among Mafia", "Mafia Town" 처럼 기존 게임 변형 | 음운·의미적으로 독립된 자체 명칭 |
| UI 화면 | 처형 화면이 특정 게임의 픽셀 구성과 동일 | 화면 분할·아이콘·전환 효과를 모두 자체 디자인 |
| 시스템 메시지 | "Town has voted" 의 직역 "마을이 결정했습니다" | 자체 카피 — 예: "광장의 결정이 내려졌습니다" |

## 6. 1단계 (M1~M3) 현재 상태 점검

지금까지 만든 코드 기준 위반 여부 자가 점검:

- 직업명: `CITIZEN / POLICE / DOCTOR / MAFIA` — 일반어, OK.
- WS 이벤트명: `assignRoles / phaseChanged / executionResult / nightResult / policeResult` — 기능 서술형, 특정 게임 고유 명칭 아님, OK.
- 페이즈명 (M4 갱신): `DAY_DISCUSSION / VOTE_SELECT / VOTE_DEFENSE / VOTE_OX / EXECUTION / NIGHT_DOCTOR / NIGHT_MAFIA / NIGHT_POLICE / NIGHT_RESULT` — 메커니즘 서술, OK.
- 게임명·앱 아이콘·스플래시·로고: **아직 미정.** 4단계 진입 전 자체 작명 + 상표 검색 필요.

## 6-1. M4 신규 룰·카피 자가점검

**메커니즘 (사회 추리 게임 일반 표준 — 사용 OK):**
- 다수결로 처형 후보 선출 후 후보의 최후 변론 시간 (모의재판 류 메커니즘).
- 광장 영역 이동으로 찬반 표시 (한국 예능 1박2일 O/X 게임 컨셉, 마피아 장르 시그니처 X).
- 처형 시 마피아 여부만 공개 (시민 추리 게임 추리 가능성 보존).

**페이즈명 (자가검증 통과):**
- `VOTE_DEFENSE` — 기능 서술형 ("최후 변론"). Town of Salem 의 `DEFENSE` 페이즈와 영문 표기만 유사하나, 모의재판 메커니즘 자체는 사회 추리 게임 장르 표준이라 페이즈명 OK. 단 UI 디자인은 Town of Salem 의 단상·아이콘 모방 X (자체 CustomPaint).
- `VOTE_OX` — 한국어권 O/X 게임 메커니즘. 어떤 특정 마피아 게임 시그니처 X.

**카피 (자체 작성 — M4 확정):**
- `"OOO님은 마피아입니다."` — 한국어 자연 카피, 특정 게임 직역 X.
- `"OOO님은 마피아가 아닙니다."` — 동일.
- `"광장의 결정 — 처형하지 않습니다."` — Town of Salem `"Town has voted"` 류와 의미·구조 다름. 자체 표현.
- `"광장이 누구도 지목하지 못했습니다."` — 동일.
- `"당신이 후보입니다 — 변론 후 광장의 결정을 기다리세요"` — 후보 본인 안내. 자체 표현.
- `"후보가 자리를 비워 광장의 결정이 무산됩니다."` — disconnect 처리 카피. 자체 표현.

**시각 디자인 (M6 작업 예정):**
- 단상: CustomPaint 자체 작화 (Town of Salem 단상 모방 X — 색·구조 차별화).
- O/X 광장: Kenney Tiny Town CC0 tile 재조합. O 영역과 X 영역 마커는 자체 작화 (특정 게임 시그니처 마크 모방 X).
- 캐릭터 동적: Puny CC0 + accessory 자체 작화. **직업별 차별화 표시 X** — 평소 익명 유지 (Town of Salem 류 직업 표시 회피).
